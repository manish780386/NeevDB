NeevDB is a lightweight, file-based database built from scratch in pure Python. No dependencies. No setup. Just pip install and go.

Getting started

Install NeevDB from PyPI. Requires Python 3.7 or newer.

Installation

# Core install (zero dependencies)
pip install neevdb

# With REST API server support
pip install neevdb[server]

# Verify
pip show neevdb

Connect to a database

Pass a file path to NeevDB(). The file is created automatically. All data is stored as human-readable JSON.

from neevdb import NeevDB

# Creates 'mydata.json' if it doesn't exist
db = NeevDB("mydata.json")

# Defaults to 'neevdb.json'
db = NeevDB()
💡The JSON file is fully human-readable. Open it in any text editor to inspect your data.

Tables

Tables hold collections of records — like sheets in a spreadsheet. Create one before inserting data.

db.create_table("users")
db.list_tables()    # → ['users']
db.drop_table("users")

CRUD operations

Create, Read, Update, Delete — the four core operations.

Insert

Add a record. Every record automatically gets _id and _created_at.

record = db.insert("users", {
    "name": "Alice",
    "age":  25,
    "city": "Mumbai"
})
# → { name: Alice, age: 25, _id: 1, _created_at: ... }

Find

# All records
db.find_all("users")

# With condition
db.find("users", lambda r: r["age"] > 18)

# Count
db.count("users")  # → 5

Update

db.update(
    "users",
    lambda r: r["name"] == "Alice",
    {"city": "Delhi", "age": 26}
)  # → 1

Delete

db.delete("users", lambda r: r["age"] < 18)
# → 1
⚠️Delete is permanent. There is no undo.

Query engine

Run SQL-like strings with db.query(). Combine WHERE, ORDER BY, and LIMIT freely.

SELECT

db.query("SELECT * FROM users")
db.query("SELECT name, city FROM users")
db.query("SELECT name FROM users WHERE age > 18 ORDER BY name LIMIT 5")

WHERE

db.query("SELECT * FROM users WHERE age > 18")
db.query("SELECT * FROM users WHERE city = Mumbai")
db.query("SELECT * FROM users WHERE age >= 21")
OperatorMeaningExample

ORDER BY

db.query("SELECT * FROM users ORDER BY name")
db.query("SELECT * FROM users ORDER BY age DESC")

LIMIT

db.query("SELECT * FROM users LIMIT 5")
db.query("SELECT name, price FROM products ORDER BY price DESC LIMIT 3")

CLI shell

Run queries from your terminal without writing Python.

Start the shell

python cli.py
python cli.py mydata.json

Commands

NeevDB > INSERT INTO users name=Alice age=25 city=Mumbai
NeevDB > SELECT * FROM users WHERE age > 18 ORDER BY name LIMIT 5
NeevDB > DELETE FROM users WHERE name = Alice
NeevDB > HELP
NeevDB > EXIT

REST API server

Start a full REST API server on top of your NeevDB database with one command.

Quick start

# Install server dependencies
pip install neevdb[server]

# Start server (opens browser automatically)
python start.py

# Or use from Python
from neevdb.server import start
start(db_path="mydata.json", port=8000)
ℹ️Once running, visit http://localhost:8000 for the dashboard and /docs for the auto-generated Swagger UI.

Use from any frontend

🚀See the API tab for full endpoint documentation with live examples.

API Reference

NeevDB exposes a full REST API when you run python start.py. Every endpoint returns JSON.

ℹ️Start the server first: python start.py — then open http://localhost:8000

All endpoints at a glance

MethodEndpointDescription
GET/apiHealth check
GET/api/statsDatabase statistics
GET/api/tablesList all tables
POST/api/tables/{name}Create a table
DEL/api/tables/{name}Drop a table
GET/api/tables/{name}/recordsGet records (with filters)
GET/api/tables/{name}/records/{id}Get single record
POST/api/tables/{name}/recordsInsert a record
PUT/api/tables/{name}/records/{id}Update a record
DEL/api/tables/{name}/records/{id}Delete a record
POST/api/queryRun SQL-like query

Base URL

All endpoints are relative to the base URL. By default:

http://localhost:8000

All responses are JSON. All POST/PUT requests must send Content-Type: application/json.

Tables

GET/api/tables

Returns a list of all table names in the database.

Response

{
  "tables": ["users", "products"]
}

Example

# curl
curl http://localhost:8000/api/tables

# JavaScript
const res  = await fetch('/api/tables')
const data = await res.json()
console.log(data.tables)  // ['users', 'products']

POST/api/tables/{name}

Create a new table. Returns 409 if table already exists.

DEL/api/tables/{name}

Permanently delete a table and all its records.

curl -X DELETE http://localhost:8000/api/tables/users
⚠️This is permanent. All records in the table will be deleted.

Records

GET/api/tables/{name}/records

Get all records from a table. Supports filtering, sorting, and limiting via query params.

Query parameters

ParamTypeDescription
wherestringFilter condition e.g. age>18 or city=Mumbai
orderstringField name to sort by e.g. name
descbooleanSort descending. Default: false
limitintegerMax records to return
 
# All records
curl http://localhost:8000/api/tables/users/records

# WHERE age > 18
curl "http://localhost:8000/api/tables/users/records?where=age>18"

# Filter + sort + limit
curl "http://localhost:8000/api/tables/users/records?where=age>18&order=name&limit=5"

# Sort descending
curl "http://localhost:8000/api/tables/users/records?order=age&desc=true"

# JavaScript
const res  = await fetch('/api/tables/users/records?where=age>18&order=name&limit=5')
const data = await res.json()
console.log(data.records)

# Response shape
{
  "table":   "users",
  "count":   3,
  "records": [{ "_id": 1, "name": "Alice", ... }]
}

GET/api/tables/{name}/records/{id}

Get a single record by its _id. Returns 404 if not found.

curl http://localhost:8000/api/tables/users/records/1

# Response
{ "record": { "_id": 1, "name": "Alice", "age": 25 } }

POST/api/tables/{name}/records

Insert a new record. Send a JSON body with a data key.

# curl
curl -X POST http://localhost:8000/api/tables/users/records \
  -H "Content-Type: application/json" \
  -d '{"data": {"name": "Alice", "age": 25, "city": "Mumbai"}}'

# JavaScript fetch
const res = await fetch('/api/tables/users/records', {
  method: 'POST',
  headers: { 'Content-Type': 'application/json' },
  body: JSON.stringify({ data: { name: 'Alice', age: 25, city: 'Mumbai' } })
})
const data = await res.json()
console.log(data.record._id)  // 1

# Response
{
  "message": "Record inserted.",
  "record":  { "name": "Alice", "age": 25, "_id": 1, "_created_at": "..." }
}

PUT/api/tables/{name}/records/{id}

Update fields of a record by _id. Only include fields you want to change.

curl -X PUT http://localhost:8000/api/tables/users/records/1 \
  -H "Content-Type: application/json" \
  -d '{"data": {"city": "Delhi", "age": 26}}'

# Response
{ "message": "Record updated.", "updated": 1 }

DEL/api/tables/{name}/records/{id}

Delete a record by _id. Returns 404 if not found.

Query endpoint

POST/api/query

Run a SQL-like query string. The most powerful endpoint — combine SELECT, WHERE, ORDER BY, and LIMIT.

GET/api/stats

Get total tables, total records, and per-table record counts.
